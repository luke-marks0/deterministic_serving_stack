What is a Manifest?

A manifest is a complete declaration of an inference run: the model, the hardware, the determinism settings, the requests, and how to compare results. Each POST /manifest is a new run — the server validates the manifest, (re)starts vLLM to match the config, and is then ready to serve.

SectionWhat it declares
modelWhich model, which exact revision, per-file digests
runtimeEngine config, batch invariance, deterministic knobs, seed
hardware_profileGPU, NIC, topology — what hardware this run requires
networkMTU, offloads, queue mapping — network determinism config
requestsThe actual prompts to serve, with temperature and token limits
comparisonHow the verifier compares two runs (exact, ULP, hash)
artifact_inputsEvery software component, content-addressed

Manifest Schema (v1)

  

    The manifest is a complete, self-describing declaration of an inference run. Every field is validated against
    manifest.v1.schema.json before the server accepts it. Click any section to expand.

          manifest_version
          const "v1"
          Schema version. Always "v1".
        

        

          run_id
          string
          Unique identifier for this run. Pattern: ^[A-Za-z0-9][A-Za-z0-9._-]{2,127}$
        

        

          created_at
          date-time

        6 required
      

      

        

          source
          string
          HuggingFace model ID. Pattern: ^hf://Org/Model$
        

        

          resolved_revision
          string (sha1)
          Pinned HF commit hash (40 hex chars). Set by resolver.
        

        

          weights_revision
          string (sha1)
          Commit that produced the weights files.
        

        

          tokenizer_revision
          string (sha1)

          boolean

          required_files
          array (min 6)
          Per-file manifest: role, path, URI, sha256 digest, size. Roles: weights_shard, config, tokenizer, generation_config, chat_template, prompt_formatter.

          enum

gpu

          

            vendor (nvidia), model, count, driver_version, cuda_driver_version, pci_ids[]
          

        

        

          
nic

          

            model, pci_id, firmware, link_speed_gbps, offloads (tso, gso, checksum, vlan_strip)

        ▶
        network
        11 required
      

      

        

          security_mode
          enum
          plaintext | tls_deterministic_test_only | tls. If TLS, egress reproducibility must be false.
        

        

          egress_reproducibility
          boolean
          Whether network egress bytes must be bitwise reproducible.
        

        

          mtu / mss
          integer
          MTU (576–9216) and MSS (536–9000) for deterministic framing.
        

        

          tso / gso / checksum_offload
          boolean
          NIC offload toggles. Deterministic mode typically disables all.
        

        

          queue_mapping
          object
          TX/RX queue counts + mapping policy (fixed_core_queue | flow_hash).
        

        

          ring_sizes
          object
          TX/RX ring buffer sizes (min 64).
        

        

          thread_affinity
          integer[]
          CPU core pinning for network threads.
        

        

          internal_batching
          object
          enabled + max_burst for packet batching.
        

      

    


    
    

      

        ▶
        requests
        min 1 item
      

      

        

          id
          string
          Unique request ID. Pattern: ^[A-Za-z0-9._:-]+$
        

        

          prompt
          string
          The input text. Min length 1.
        

        

          max_new_tokens
          integer (1–4096)
          Maximum tokens to generate.
        

        

          temperature
          number (0–2)
          Sampling temperature. 0 = greedy.
        

      

    


    
    

      

        ▶
        comparison
        4 comparators
      

      

        
Defines how the verifier compares two run bundles. Each comparator has a mode:

        

          

            exact
            Bitwise identical
          

          

            hash
            SHA256 match (algorithm required)
          

          

            ulp
            Units in Last Place tolerance (ulp required)
          

          

            absrel
            Absolute + relative tolerance (atol, rtol required)
          

        

        

          Applied to: tokens, logits, activations, network_egress
        

      

    


    
    

      

        ▶
        artifact_inputs
        min 10 items
      

      

        
Every software component is tracked as a content-addressed artifact. Minimum 10 artifacts covering the full stack:

        

          model_weights
          model_config
          tokenizer
          serving_stack
          cuda_lib
          kernel_library
          network_stack_binary
          pmd_driver
          runtime_knob_set
          request_set
          batching_policy
          nic_link_config
          collective_stack
          compiled_extension
          remote_code
        

        

          Each artifact has: artifact_id, artifact_type, source_kind (hf|oci|s3|http|git|nix|inline), source_uri, immutable_ref, and optional expected_digest.
        

      

    


    
    

      

        ▶
        deterministic_dispatcher
        conditional
      

      

        
Required when topology mode is replicated, tensor_parallel, or pipeline_parallel.

        

          enabled
          boolean
          Must be true for multi-node topologies.
        

        

          algorithm
          enum
          round_robin_hash | sequence_map
        

        

          request_order_source
          enum
          ingress_sequence — order determined by arrival.
        

        

          replay_log_required
          boolean
          Whether dispatch decisions must be logged for replay.
